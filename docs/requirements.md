## 一、项目概述

### 1.1 产品定位

萌宠英语乐园是一款面向**考研/四六级水平英语学习者**的**中英互译拼写游戏**，以萌宠培育为游戏驱动力，融合游戏化学习体验。

### 1.2 目标用户

- 大学英语四级、六级备考学生
- 考研英语备考学生
- 对游戏化学习感兴趣的英语学习者

### 1.3 核心玩法闭环

答题获得积分 → 积分兑换食品/玩具 → 投喂/玩耍培育萌宠 → 萌宠陪伴提升学习动力 → 继续答题

### 1.4 视觉风格

- 主色调：天蓝 #87CEEB、淡蓝 #B0D4F1、青绿 #7EC8E3
- 辅助色：白色 #FFFFFF、淡紫 #E8E0F0
- 整体感觉：清新、青春、治愈、圆润卡片设计

## 三、首页模块

### 3.1 功能描述

首页是用户进入游戏后的主界面，集中展示宠物状态和所有功能入口。

### 3.2 界面元素

| 元素 | 说明 |
|------|------|
| 宠物头像 | 已选宠物的头像图片与昵称展示；未选宠物时显示默认占位图 |
| 状态栏 | 生命值 ❤️、答题数 🌱、连击 🔥、积分 ⭐（四栏图标均分显示） |
| 每日一句 | 显示一句励志/英语相关的每日名言（硬编码展示） |
| 操作入口 | 开始答题、宠物屋、积分商城、选择萌宠、错题本 |

### 3.3 用户操作流程

1. 用户打开页面 → 自动加载首页
2. 若未选择宠物，页面显示"选择萌宠"入口，引导用户选蛋
3. 点击「开始答题」→ 进入答题页面
4. 点击「宠物屋」→ 进入宠物互动页面
5. 点击「积分商城」→ 进入商城页面
6. 点击「选择萌宠」→ 进入选蛋/重新选择宠物页面
7. 点击「错题本」→ 进入错题复习页面

## 四、选蛋/萌宠选择模块

### 4.1 功能描述

首次进入游戏时引导用户选择一只宠物，确认后绑定为用户伙伴。后续可重新选择。

### 4.2 可选宠物

| 宠物 | 标识 | 素材文件 |
|------|------|---------|
| 🐱 猫 | cat | 小猫.jpg |
| 🐶 狗 | dog | 狗.webp |
| 🐹 豚豚 | 豚豚 | 豚豚.webp |
| 🦝 噜噜 | 噜噜 | 噜噜.webp |
| 🐕 线条小狗 | 线条小狗 | 线条小狗(male).jpeg / 线条小狗(female).jpeg |

### 4.3 用户操作流程

1. 展示 5 种宠物卡片供用户浏览
2. 点击选择一种宠物 → 弹出命名弹窗
3. 输入宠物昵称（必填）
4. 确认选择 → 更新用户数据，宠物绑定成功
5. 跳转至首页，显示已选宠物

### 4.4 数据说明

- 已选宠物信息存储在 `userData.pet` 对象中
- 包含宠物类型、昵称、表情符号、心情值和饱腹度初始值（均为 100）
- 标记 `hasChosenPet: true`

## 五、答题系统模块

### 5.1 功能描述

核心玩法模块。用户进行中英互译拼写答题，系统提供题目、判定答案、管理生命值和积分。

### 5.2 出题机制

| 模式 | 说明 | 数据来源 |
|------|------|---------|
| 本地题库模式（默认） | 从预置题库中随机抽取 | `gameData.js` + `imported_bank.js`（100+ 词汇/短语/句子） |
| AI 实时出题模式（可选） | 调用大模型 API 动态生成题目 | 由 `aiService.js` 中的 `aiGenerateQuestion()` 实现 |

- 每局出题数量：**12 题**
- 题目随机混合三种难度和三题型
- 用户可通过开关切换出题模式

### 5.3 难度系统

| 难度 | 词汇水平 | 标识 |
|------|---------|------|
| 🌱 EASY — 四级 | 大学英语四级词汇 | easy |
| 🌿 MEDIUM — 六级 | 大学英语六级词汇 | medium |
| 🌳 HARD — 考研 | 考研英语词汇 | hard |

- 每局游戏随机混合三种难度
- 题目类型包含：单词、短语、句子

### 5.4 题型

| 题型 | 标识 | 说明 |
|------|------|------|
| 中文→英文 | cn2en | 显示中文，用户拼写英文 |
| 英文→中文 | en2cn | 显示英文，用户拼写中文 |

### 5.5 答题交互

- **输入方式**：文本输入框
- **提交答案**：点击「提交答案」按钮
- **查看提示**：点击「提示」按钮展示 hint 字段
- **跳过/不认识**：点击「不认识」按钮，展示正确答案，扣 1 条命，记录到不认识列表
- **答对**：显示 ✅ 反馈，1.5 秒后自动跳转下一题
- **答错**：显示 ❌ 反馈及正确答案，需手动点击下一题
- **连对 3 题**：奖励 1 条命（不超过上限 10 条），显示连击横幅
- **可中途退出**：退出时保存答题进度（已答题数、积分、生命值），下次进入可继续

### 5.6 生命值机制

| 项目 | 值 |
|------|-----|
| 初始生命值 | 10 条命 |
| 最大生命值 | 10 条命 |
| 答错惩罚 | -1 条命 |
| 跳过惩罚 | -1 条命 |
| 连对 3 题奖励 | +1 条命（不超过 10 条） |
| 生命归零 | 游戏结束，跳转结算页；结算后生命值重置为 10 条 |

### 5.7 积分机制

| 题目难度 | 答对得分 |
|---------|---------|
| EASY（四级） | +10 分 |
| MEDIUM（六级） | +20 分 |
| HARD（考研） | +30 分 |

- AI 评测裁定为正确后，补加对应分数

### 5.8 答题结算页

答题结束后展示：

- 游戏统计数据（得分、正确/错误数量、最高连击数）
- 每道题答题历史记录（题目、用户答案、是否正确、正确答案）
- 语法纠错反馈展示

### 5.9 错题自动收录

- 答错的题目自动收录到错题本
- 收录数据结构包含：题目 ID、中英文、题型、难度、词性、错误次数、收录时间

## 六、AI 评测模块

### 6.1 功能描述

对用户的英文→中文翻译答案进行**语义级智能评测**，而不仅仅是精确字符串匹配。支持开关切换。

### 6.2 评测方式

| 评测方式 | 优先级 | 说明 |
|---------|--------|------|
| AI API 评测 | 优先 | 调用大模型 API，由 AI 判断语义相似度 |
| 本地 NLP 回退 | 备选 | 无 API Key 或 API 失败时自动回退 |

### 6.3 精确匹配（基础判定）

- 中文→英文题型：始终使用**精确字符串匹配**（忽略大小写、去空白）
- 英文→中文题型：先尝试精确匹配，匹配成功直接判对

### 6.4 AI API 评测

- 构建评测 Prompt，包含翻译方向、源文、用户答案、参考答案
- 调用大模型 API，解析 JSON 格式的评测结果 `{"isCorrect": bool, "reason": string}`
- 评测规则：
  - 语法正确率 ≥ 95% 算通过
  - 逻辑正确率 ≥ 80% 算通过
  - 英译中：核心意思一致就算对
  - 中译英：允许合理同义替换
- 异步回调：AI 评测完成后补正判定结果和分数
- UI 展示 AI 评测状态：「🤖 AI 正在评测语义相似度...」「✅ AI：语义相似」「❌ AI：语义不匹配」

### 6.5 本地 NLP 回退评测

当无 API Key 或 API 调用失败时，使用本地算法：

- **语法检查**：检测中英文混写、重复字符等问题，计算语法正确率
- **分词相似度**：基于自定义词典的 Jaccard 相似度计算
- **编辑距离**：Levenshtein 编辑距离归一化相似度
- **同义词匹配**：预定义 13 组中文同义词（如 高兴/开心/快乐），整词边界匹配
- **综合评分**：Jaccard × 0.6 + 编辑距离相似度 × 0.4，≥ 0.4 或命中同义词即判对

### 6.6 开关控制

- 用户可通过「AI 评测」开关切换 AI 评测/精确匹配模式
- 开关状态实时切换，影响下一题提交时的判定方式

## 七、AI 出题模块与 API 调用

### 7.1 功能描述

支持接入大模型 API 实现实时 AI 出题和 AI 语义评测。用户可配置 API 服务商和 Key。

### 7.2 支持的服务商

| 服务商 | 配置标识 | 基础 URL | 模型 |
|--------|---------|---------|------|
| 阿里云通义千问（百炼平台） | tongyi | https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions | qwen-turbo |
| OpenAI | openai | https://api.openai.com/v1/chat/completions | gpt-4o-mini |
| DeepSeek | deepseek | https://api.deepseek.com/v1/chat/completions | deepseek-chat |
| 硅基流动（SiliconCloud） | siliconflow | https://api.siliconflow.cn/v1/chat/completions | Qwen/Qwen2.5-7B-Instruct |

### 7.3 API Key 管理

- 用户在游戏页面中通过下拉菜单选择服务商，输入 API Key
- API Key 存储在 `localStorage`，键为 `{provider}ApiKey`（如 `tongyiApiKey`）
- 支持在 `aiService.js` 中的 `AI_CONFIG` 区域直接填写默认 Key
- 页面提供保存成功状态提示

### 7.4 AI 出题功能

- 开关控制：「✍️ AI 出题已开启」/「📖 使用本地题库」
- 开启动态生成题目，替代从本地题库抽取
- 出题 Prompt 要求：随机选择难度和题型，标注词性，提供 hint
- 返回 JSON 格式题目，与本地题目数据结构兼容
- 可配置出题参数：`max_tokens: 200`、`temperature: 0.8`
- 超时控制：10 秒超时，失败时静默回退到本地题库

### 7.5 AI 评测 API 调用

- 可配置评测参数：`max_tokens: 200`、`temperature: 0.3`、`timeout: 15000`
- 与 AI 出题共用服务商选择

### 7.6 API 配置界面

- 在游戏页面中提供 API 设置面板（可折叠）
- 设置项：服务商下拉选择、API Key 输入框、保存按钮、状态提示

## 八、宠物屋模块

### 8.1 功能描述

查看和管理宠物状态，通过投喂和玩耍与宠物互动。

### 8.2 宠物属性

| 属性 | 范围 | 说明 |
|------|------|------|
| 心情值 (happiness) | 0~100 | 通过玩耍提升 |
| 饱腹度 (fullness) | 0~100 | 通过投喂提升 |
| 初始值 | 均为 100 | 新选/重置宠物时 |

### 8.3 状态衰减机制

- 每 **10 分钟**心情值和饱腹度各衰减 **5 点**
- 最低降至 0
- 使用 `setInterval` 实现定时衰减
- 如果当前在宠物屋页面，衰减后自动刷新显示

### 8.4 用户操作

| 操作 | 效果 | 消耗 |
|------|------|------|
| 投喂食物 | 提升饱腹度 | 消耗背包中的食品 |
| 玩耍玩具 | 提升心情值 | 消耗背包中的玩具 |

- 投喂和玩耍后对应物品从背包中移除
- 背包为空时提示无可用物品

### 8.5 界面展示

- 宠物大图展示
- 宠物名称和类型标识
- 心情值进度条（含数字百分比）
- 饱腹度进度条（含数字百分比）
- 投喂按钮、玩耍按钮
- 背包为空时的提示

## 九、积分商城模块

### 9.1 功能描述

用户使用答题获得的积分购买食品和玩具，用于宠物屋中的互动。

### 9.2 商品分类

| 分类 | 说明 | 数量 |
|------|------|------|
| 🍖 食品 (food) | 用于投喂宠物 | 6 种 |
| 🎾 玩具 (toys) | 用于陪宠物玩耍 | 6 种 |

### 9.3 用户操作流程

1. 进入商城，查看食品/玩具分页
2. 浏览商品（名称、图标、价格、库存）
3. 点击「购买」→ 判断积分是否足够 → 扣除积分 → 减少库存 → 添加至背包
4. 积分不足时提示「积分不足！还需要 X 分」
5. 购买成功时 Toast 提示

### 9.4 库存机制

- 每种商品有独立库存数量
- 购买后库存减 1，最低为 0
- 提供「刷新库存」功能，重置所有商品库存到初始值

### 9.5 背包系统

- 独立存储用户的食品、玩具背包
- 首页状态栏展示持有积分
- 宠物屋使用时从背包扣除物品

## 十、错题本模块

### 10.1 功能描述

自动收录用户答错的题目，提供集中复习功能。

### 10.2 数据收录

- 答错时自动将题目加入错题本
- 已存在错题本中的题目，错误次数 +1
- 数据结构：题目 ID、中英文、题型、难度、词性、提示、音标、错误次数、收录时间
- 独立存储键：`petGameWrongBank`
- 「不认识」功能不会将题目加入错题本，而是加入独立的不认识列表（`unknownQuestions`）

### 10.3 展示与操作

- 以列表形式展示所有错题
- 每项显示：单词/短语、音标、正确答案、难度标签、词性、错误次数
- 提供「清空错题本」功能（需确认弹窗）

## 十三、数据模型

### 13.1 用户数据（`petGameUserData`）

```javascript
{
  pet: {
    type: 'cat',           // 宠物类型标识
    name: '咪咪',           // 用户自定义昵称
    emoji: '🐱',            // 表情符号
    happiness: 100,         // 心情值 0~100
    fullness: 100           // 饱腹度 0~100
  },
  hasChosenPet: true,       // 是否已选择宠物
  lives: 10,                 // 当前生命值
  maxLives: 10,              // 最大生命值
  streakCount: 0,           // 当前连对计数
  totalCorrect: 0,          // 累计答对数
  totalWrong: 0,            // 累计答错数
  score: 0,                 // 当前积分
```
