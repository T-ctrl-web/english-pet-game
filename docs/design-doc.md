## 一、项目概述

一款面向考研/四六级水平英语学习者的**中英互译拼写游戏**，以萌宠培育为驱动力，融合游戏化学习体验。

### 核心定位
- **目标用户**：考研/四六级水平的英语学习者
- **学习方式**：中英互译拼写输入
- **风格调性**：淡色青春风（天蓝、淡蓝、青绿）
- **驱动力**：萌宠培育 + 积分兑换

## 二、核心玩法系统

### 2.1 出题机制
| 模式 | 说明 |
|------|------|
| 基础模式（默认） | 使用本地预置题库，无需网络请求 |
| AI增强模式（可选） | 接入 DeepSeek API 实时生成题目（后续实现） |

**基础版以本地题库为主**，避免频繁 API 调用产生费用。

### 2.2 难度系统
| 难度等级 | 词汇水平 | 示例 |
|---------|---------|------|
| 🌱 EASY - 四级 | 大学英语四级词汇 | beautiful → 美丽的 |
| 🌿 MEDIUM - 六级 | 大学英语六级词汇 | phenomenon → 现象 |
| 🌳 HARD - 考研 | 考研英语词汇 | paradoxical → 悖论的 |

- 每局游戏**随机混合**三种难度
- 题目类型包含**单词、短语、句子**

### 2.3 答题规则
- **题型**：纯拼写输入（中文→英文 / 英文→中文）
- **无固定答案**：语义匹配判定
- **语法判定（英译中）**：语法正确率 ≥ 95% 算通过
- **逻辑判定（英译中）**：逻辑相似度 ≥ 40% 算通过
- **中文→英文**：精确匹配
- **答题完成**：展示详细的语法纠错和优化建议

### 2.4 生命机制
- 初始生命：**5 条命**
- 答错：扣除 1 条命
- **连对 3 题**：奖励 1 条命（上限 5 条）
- 生命归零：游戏结束

### 2.5 答题流程
- 答对后 1.5 秒自动跳转下一题
- 答错后手动点击下一题
- 中途退出保留已获得积分

## 三、萌宠培育系统

### 3.1 初始选择
新手首次进入时，从以下 **5 种宠物蛋**中随机选择一个：

| 宠物 | 风格参考 | 图片素材 |
|------|---------|---------|
| 🐱 猫 | 真实猫咪照片 | 小猫.jpg |
| 🐶 狗 | 简约卡通狗 | 狗.webp |
| 🐹 豚豚 | 圆滚滚仓鼠 | 豚豚.webp |
| 🦝 噜噜 | 卡通小浣熊 | 噜噜.webp |
| 🐕 线条小狗 | 简约线条风格 | 线条小狗(male).jpeg |

### 3.2 成长机制
- 答题获得**积分**
- 积分用于在**商城**兑换：食品、玩具
- 宠物状态：心情（玩耍提升）、饱腹（投喂提升）
- 不同宠物有不同风格和互动反馈

### 3.3 后续扩展
- 定期推出新宠物
- 宠物稀有度系统
- 宠物养成等级

## 四、UI/UX 设计

### 4.1 视觉风格
- **主色调**：天蓝 #87CEEB、淡蓝 #B0D4F1、青绿 #7EC8E3
- **辅助色**：白色 #FFFFFF、淡紫 #E8E0F0
- **字体风格**：圆润、轻松
- **整体感觉**：清新、青春、治愈

### 4.2 主要页面
1. **首页** — 萌宠展示 + 开始答题入口 + 每日一句
2. **选蛋页** — 首次进入时的宠物选择（弹窗取名）
3. **答题页** — 题目展示 + 输入框 + 生命/积分/进度/反馈
4. **结算页** — 答题报告 + 语法纠错
5. **宠物屋** — 宠物状态 + 投喂/玩耍互动
6. **商城** — 积分兑换食品/玩具（含库存管理）

## 五、技术方案（Web 版）

### 5.1 技术栈
| 层级 | 技术选型 |
|------|---------|
| 前端框架 | 原生 HTML + CSS + JavaScript |

## 六、API 与费用考量

### 基础版策略
- **默认使用本地题库**，零 API 费用
- AI 出题作为**可选增强**，由用户手动开启（后续实现）
- AI 语义评测使用本地 NLP 算法，完全免费
- 本地题库覆盖 100+ 词汇/短语/句子

### 上架后策略
- 用户答题**不会**消耗开发者的 API 额度
- 方案一：自建后端服务器，统一管理 API Key
- 方案二：使用用户自带 API Key（高级功能）
